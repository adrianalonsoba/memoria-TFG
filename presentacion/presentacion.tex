\begin{frame}\frametitle{Dashboards en 2D } 
{\LARGE 
\begin{exampleblock}{Ventajas}
\begin{enumerate}
\item Simplicidad de representación.
\item No requieren aceleración gráfica.
\item Impacto mínimo en el rendimiento.
\end{enumerate}
\end{exampleblock}

\begin{alertblock}{Inconvenientes}
\begin{enumerate}
\item No se pueden representar gráficos con mas de 2 dimensiones, por razones obvias.
\item Las posibilidades de visualización y colocación son limitadas.
\end{enumerate}
\end{alertblock}
}
\end{frame}


\subsection{Objetivo principal}
\begin{frame}
\frametitle{Objetivo principal} 
{\large 
Crear una librería que nos permita crear visualizaciones y filtrar datos de desarrollo de software en 3D, dentro de cualquier navegador.
\begin{block}{Requisitos}
\begin{enumerate}
\item Conseguir una interfaz y funcionalidad tan parecidos a los de dc.js como sea posible.
\item Tener varios tipos de gráficas y ser capaces de filtrar.
\item Utilizar un framework de webGL
\item Respuesta rápida de los filtros.
\item Debemos ser capaces de hacer zoom, desplazarnos y arrastrar las gráficas.
\item Debemos poder colocar gráficas tanto de forma independiente como dentro de paneles.
\item Tener una estructura basada en programación orientada a objetos.
\end{enumerate}
\end{block}
}
\end{frame}

\subsection{Funciones principales}
\begin{frame}
\begin{block}{Métodos en cadena}
\begin{itemize}
\item \textbf{group}:Establece el grupo de Crossfilter.
\item \textbf{dimension}:Establece la dimension de Crossfilter.
\item \textbf{width}: El ancho de la gráfica.
\item \textbf{height}: El alto de la gráfica.
\item \textbf{gridsOn}: Activa los grids
\item \textbf{gridsOff}: Descactiva los grids.
\item \textbf{numberOfXLabels}: Establece el numero de etiquetas del eje x.
\item \textbf{numberOfYLabels}: Establece el numero de etiquetas del eje y.
\item \textbf{color}: Establece el color de la gráfica.
\item \textbf{depth}: Establece la profundidad de la gráfica.
\item \textbf{opacity}: Establece la la opacidad de la gráfica
\end{itemize}
\end{block}
\end{frame}
\subsection{Tipos de gráficas}
{\LARGE 
\lstset{language=Java, breaklines=true, basicstyle=\footnotesize}
Una vez creada la escena con las condiciones de camara y luz deseadas, hacemos uso de la librería:
\begin{lstlisting}[frame=single]
var line =  THREEDC.lineChart([0,0,0]);
line.group(groupByMonth)
    .dimension(dimByMonth)
    .width(500)
    .numberOfXLabels(10)
    .numberOfYLabels(15)
    .gridsOn()
    .height(200)
    .color(0x0000ff)
    .depth(20);
THREEDC.renderAll();
\end{lstlisting}
}

\subsection{Paneles}
{\Large 
\begin{block}{Paneles}
Los \textbf{paneles} tienen como objetivo contener las gráficas relacionadas, sus parámetros son:
\begin{enumerate}
\item coords
\item numberOfCharts
\item size
\end{enumerate}
\end{block}

Un ejemplo de instanciación de panel con 3 gráficas:

\begin{lstlisting}[frame=single]
var panel=THREEDC.addPanel([0,0,0],3,[200,200]);
\end{lstlisting}

\section{Conclusiones}
Código para crear la misma gráfica de tarta usando dc.js y THREEDC.js:
Usando dc.js:
\begin{lstlisting}[frame=single]
//HTML5 div to attach the pie
var pieDC = dc.pieChart("#test");

pieDC.radius(100)
	 .dimension(dimPerOrg)
	 .group(groupPerOrg);
dc.renderAll();
\end{lstlisting}
Usando THREEDC.js:
\begin{lstlisting}[frame=single]
//Coordinates to attach the pie, a panel could be used too.
var pieTHREEDC =  THREEDC.pieChart([0,0,0]);

pieTHREEDC.radius(100)
          .dimension(dimPerOrg)
          .group(groupPerOrg);

THREEDC.renderAll();
\end{lstlisting}
